Answer I got:

==========
Congratulations on finishing Chapter 5! You have reached a major milestone. You now understand Ownership, Borrowing, Slices, and Structs with Methods. This is the "core" of Rust.

Here are 5 practice homework assignments designed to test these specific skills. They start with basic struct usage and end with complex ownership and slice management.

1. The Grade Tracker (Basic Structs & Methods)

Goal: Create a struct and use basic &self methods.

Create a struct Student with fields: name (String), score (u32).

Implement a method is_passing(&self) -> bool that returns true if the score is 70 or higher.

Implement a method give_extra_credit(&mut self, amount: u32) that increases the score (clamped at 100).

Bonus: In main, create a vector of 3 students and use a loop to print which ones passed.

2. The Digital Wallet (Mutability & Logic)

Goal: Master &mut self and conditional logic inside methods.

Create a struct Wallet with a field balance (f64).

Implement an associated function (static method) new(start_balance: f64) -> Wallet to initialize it.

Implement deposit(&mut self, amount: f64).

Implement withdraw(&mut self, amount: f64) -> bool. It should only deduct money if the balance is sufficient and return true. If insufficient, return false and don’t change the balance.

Constraint: Ensure your program handles the case where someone tries to withdraw a negative amount.

3. The Rectangle Overlap (Interaction Between Structs)

Goal: Learn how to pass one struct into another struct's method by reference.

Create a struct Rectangle with width and height (u32).

Implement a method area(&self) -> u32.

Implement a method can_hold(&self, other: &Rectangle) -> bool. This should return true if the current rectangle is strictly larger in both width and height than the other rectangle.

Challenge: Implement a method scale(&mut self, factor: u32) that multiplies both dimensions.

4. The Library System (Ownership & References)

Goal: Understand how structs handle owned Strings and how to compare them.

Create a struct Book with title (String) and is_available (bool).

Implement a method borrow_book(&mut self). If the book is available, set it to false and print "Success". If not, print "Already out".

Implement a method return_book(&mut self).

Implement a method matches_title(&self, search: &str) -> bool. Use this to check if a search string matches the title.

Question to solve: In main, if you move a Book into a function, can you still use it in main? Prove it with code.

5. The "Text Processor" (Slices & Structs - The Final Boss)

Goal: Combine Chapter 4 (Slices) with Chapter 5 (Structs). This is the hardest.

Create a struct Document that holds a content field (String).

Implement a method first_word(&self) -> &str. This method should return a slice to the first word of the content (search for the first space, just like the example in Chapter 4.3).

Implement a method word_count(&self) -> usize. (Hint: use .split_whitespace() and .count()).

The Crucial Test: In main:

Create a Document instance.

Call first_word and save the result in a variable word.

Try to call a &mut self method (like clear_content) while word is still in use.

Observe the compiler error. This confirms you understand how the Borrow Checker keeps your structs safe!

Quick Check: What should you know by now?

Before you start these, ensure you can answer these three questions:

Why do we use &self instead of self in a method?

What is the difference between an associated function (like String::from) and a method (like .len())?

If a struct contains a String, and you assign that struct to a new variable, what happens to the data? (Hint: Move).
